HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • the title element defines the title for the webpage and is very important for SEO.
  • SEO stands for search engine optimization.
  • style element defines the CSS styles associated with a website.
  • Base element defines the base URL for a webpage.
  • Link element connects an external resource to the HTML document.
  • Meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • Script element embeds client-side scripts in the HTML document.
  • The closing head tag looks like .
  • Child elements are nested insdie other elements.
  • Parent elements hold all the child elements.
  • The name of a sematic element describes its content and the role it plays on the webpage.
  • A tag refers only to what appears inside the angle brackets.
  • UL element is an unordered list.
  • OL element is an ordered list.
  • li element is a list of items
  • Img element contains info about images that are displayed on the webpage.
  • SRC attribute defines the location of an image.
  • Br element creates a line of empty space or a line break.
  • Herf attribute points to the URL for the link.
  • Id attribute a way to identify elements and select them.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is when a developer wants to style an element directly in the HTML file.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet allows developers to keep all their CSS rules in a separate file.
  • A selector defines the element or attribute to which the rules, or declarations, will apply.
  • Use this specail wild card symbol * to apply a rule to all elements visible on the page.
  • DRY code stands for Do Not Repeat Yourself.
  • Test-align is a positioning property that can help center a title or heading.
  • An img selector assigns property vaules to any image element in the markup.
  • A link element creates a connection to an external file.
  • A rel attribute specifies the relationship between the current documentation and the linked document resource.
  • A href attribute specifies the location URL of the external resource.
  • box-shadow CSS property adds a shadow effect around an elements frame.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repo or repository is a folder in which all files and their versions history is stored.
  • A branch is a workspace in which you can make changes that wont affect the live site.
  • Markdown is a way to write in github that converts plain text to github code.
  • Commit changes is a saved record of changes made to a file within the repo.
  • Merge after a pull request is approved, the commit wil be pulled in from one branch to another and then, deployed on the live site.
  • Issues are a way work is tracked in Git.
  • A pull request or PR is a way to ask for changes made to another branch to be merges into another branch that also allows for multiple users to see.
  • IDE is an Intergrated development environment.
  • A local environment is the workspace where each developer does their work independently on their own computer.
  • Git checkout main with bring you back to the main brach.
  • Git pull origin main is a pull request that receives a branchs modifications into a local branch.
  • Git push origin feature/add-js is a push to put you new changes in the repo.
  • git add -A is how you stage new files after saving changes.
  • git commit -m is used to comment on the changes you made while working on the feature branch.
  • Cd is used to change directories in git bash.
  • Ls will list all files instead a certain directory.
  • Code . will pull up VS code.
  • Cd . . with bring you back to the previous directory.
